# easy-capture — PoC 검증 계획

> 최종 업데이트: 2026-05-27 · 관련: [RFP](RFP.md) · [데이터플로우](data-flow.md)
> 목적: 본격 구현 전에 **가장 큰 기술 리스크를 코드로 검증**하고, 설계 파라미터를 보정한다.

---

## 1. 검증 가설 (코드 실행)

| ID  | 가설                                                                            | 리스크                |
| --- | ----------------------------------------------------------------------------- | ------------------ |
| H1  | 공식 `sam2` 패키지 video predictor 를 로컬에 설치·실행할 수 있고, 군무 클립에서 선택 인물을 일정 수준 추적 유지한다 | 높음 (성숙도·설치)        |
| H2  | PySceneDetect 컷 감지 + 위치/클래스 재매칭으로 **컷을 넘어 동일인 재추적**이 현실적으로 동작한다               | **최상 (핵심 제품 리스크)** |
| H3  | CPU 전용 환경에서 짧은 구간 추적이 "사용 가능한" 대기시간 내 완료된다                                    | 중간                 |
| H4  | PyAV 로 VFR 영상의 프레임·오디오 PTS 동기를 맞춰 MP4 를 무손실 mux 할 수 있다                        | 중간                 |

> 라이선스(basicsr/Real-ESRGAN/FFmpeg 빌드)는 **코드 PoC 아님** — [resources.md](resources.md) §2 에 문서로 검증·기록.

---

## 2. 테스트 데이터셋 (스펙 명시)

| 샘플 | 스펙 | 검증 대상 |
|---|---|---|
| A. 군무 MV 클립 | 1080p, ~30초, 컷 ~5회, 인물 4~6명 | H1, H2 (identity 유지·컷 재추적) |
| B. 빠른 컷 클립 | 1080p, ~15초, 컷 ~10회 | H2 (재매칭 한계) |
| C. 직캠 저화질 | 720p, 노이즈 높음, 단일 인물 | H1 (저화질 추적), 입력검증 경고 |
| D. VFR 영상 | 가변 fps 소스 | H4 (오디오 동기) |

> 샘플은 권리상 사용 가능한 영상으로 준비하고, 각 샘플의 **출처·해상도·fps·컷 수**를 PoC 리포트에 기록한다.

---

## 3. 정량 성공 기준 (KPI)

| KPI | 목표 | 측정 |
|---|---|---|
| 추적 유지율 (A) | ≥ 80% 프레임 | GT 마스크/박스 대비 IoU ≥ 0.5 비율 |
| occlusion 재추적 성공 | ≥ 70% | 대기시간 내 재등장 시나리오 통과율 |
| 컷 재매칭 성공 (A,B) | ≥ 70% | 컷 직후 동일인 정확 재매칭 비율 |
| GPU 처리 속도 | ≥ 10 fps | 프레임수/처리시간 |
| CPU 처리 (C, 짧은 구간) | 안내 시간 내 완료 | 사용성 판정 |
| 오디오 동기 (D) | 어긋남 ≤ 1프레임 | 파형 정렬 확인 |

미달 시: 모델 티어 상향, 재매칭 가중치/임계값 조정, 대기시간 기본값 조정, 또는 해당 기능 범위 재조정.

---

## 4. 보정할 설계 파라미터

- 샷 재매칭 점수 가중치 `w_pos`(기본 0.7) / `w_cls`(기본 0.3), 임계값(기본 0.5).
- occlusion 대기시간 기본값(프레임/초).
- 떨림 완화 이동평균 N(기본 5).
- 디바이스별 SAM2 기본 티어 매핑.
