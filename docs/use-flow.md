## 0. 앱 초기화 (첫 실행)

```
앱 실행
  └─ 모델 캐시 확인
       ├─ 캐시 있음 → 시작 화면으로
       └─ 캐시 없음 → 모델 다운로드 (SAM2 / Grounding DINO / 업스케일러)
                        진행률·ETA 표시, 실패 시 재시도/오프라인 안내
  └─ 디바이스 감지 (CUDA / CPU) → 모델 티어·예상 처리시간 결정
  └─ 저작권 이용 고지 1회 표시
```

상세 에러 분기는 [error-handling.md](error-handling.md).

---

> **입력 안내 (모든 모드 공통)**: 입력 검증 시 ① 저화질(<480p) → 경고+업스케일 권유, ② **직캠(광각/세로 팬캠) → "화면 모서리 추적 정확도가 낮을 수 있어요" 안내**(왜곡 보정 v1.1), ③ **VFR(가변 프레임) 감지 → "오디오 동기가 부정확할 수 있어요. MP4 는 지정 FPS 로 정규화됩니다" 고지, 동기 실패 시 무음 MP4 폴백**.

## 1. 모드 선택 (기능 분리)

시작 화면에서 둘 중 하나 선택:

- **① 이미지(짤) 모드** — 한 장의 정지 이미지를 잘라낸다.
- **② GIF(움짤) 모드** — 구간을 추적해 GIF/MP4 를 만든다.

> 모드는 출력 직전이 아니라 **진입 시점에** 분리한다. 이후 UI·흐름이 모드별로 달라진다.

## 2. 이미지(짤) 모드 시나리오

```
입력(영상 또는 사진)
  └─ 입력 검증 (해상도/코덱) → 권장 미만이면 경고
  └─ [영상이면] 타임라인에서 시점 선택
  └─ 세그멘테이션 실행 (Grounding DINO 검출 → SAM2 마스크)
       └─ 동일 프레임 다중 인물 → 후보 박스·라벨 표시
  └─ 사용자가 대상 1개 클릭 확정 (빈 곳 클릭 시 SAM2 직접 프롬프트)
  └─ 크롭 범위(W×H) + 종횡비 프리셋 지정
  └─ [옵션] 업스케일 (모델·배율 선택)
  └─ 미리보기
  └─ PNG/JPG 내보내기
```

## 3. GIF(움짤) 모드 시나리오

```
입력(영상)
  └─ 입력 검증
  └─ 시점 선택 → 세그 → 후보 클릭 확정 (이미지 모드와 동일)
  └─ 크롭 범위(W×H) + 종횡비 프리셋
  └─ 타임라인 드래그로 [시작]~[끝] 구간 지정
  └─ GIF 설정: FPS · 재등장 대기시간 · 갭 채우기 방식 · 팔레트 품질
  └─ [옵션] 업스케일
  └─ 추적 실행 ──────────────┐  (워커 스레드, 진행률·ETA)
  └─ 미리보기 (재생)          │
       ├─ 만족 → 내보내기      │
       └─ 이상 → 수동 교정 ────┘ (§5)
  └─ GIF 또는 MP4(오디오 포함) 내보내기
```

### 3-1. 추적 중 분기 (자동)

```
프레임 전파 중
  ├─ 샷 경계(컷) 감지 (PySceneDetect)
  │     └─ 컷 직후 재검출 → 동일인 재매칭 점수 계산
  │          ├─ 점수 ≥ 임계값 → SAM2 재초기화·추적 계속
  │          └─ 점수 < 임계값 → 사용자에게 "이 사람이 맞나요?" 확인
  │
  └─ 오브젝트 소실(마스크 신뢰도 저하) → occlusion 대기
        ├─ 대기시간 내 재등장 → 재추적 계속
        └─ 대기시간 초과 → 구간 종료
              └─ 갭 출력은 선택한 방식으로:
                   ① 컷: 갭 프레임 제외
                   ② 배경 계속: 마지막 크롭 위치 고정해 배경 출력
                   ③ 프리즈: 마지막 크롭 프레임 복제
```

## 4. 다중 멤버 (순차 처리, MVP)

```
멤버 A 추적·내보내기 완료
  └─ "같은 구간에서 다른 멤버도?" → 예
       └─ 세션·디코드 캐시 유지 (영상 재로드·재디코드 없음)
       └─ 시점에서 다시 세그 → 멤버 B 클릭 → 범위 → 추적 → 내보내기
  └─ 반복
```

> 일괄 배치(동시 입력·병렬 처리)는 v1.1. MVP 는 캐시 재사용으로 반복 비용을 줄인다.

## 5. 수동 교정 (부분 재추적)

```
미리보기 재생 중 "이 구간 이상" 발견
  └─ 해당 지점에서 일시정지
  └─ "교정 모드" 진입
       └─ 현재 프레임에 세그 오버레이 재실행
       └─ 올바른 오브젝트 클릭(또는 박스 드래그)로 재지정
       └─ "이 지점부터 재추적" 실행
            └─ 이전 성공 구간은 보존·재사용, 이 프레임 이후만 다시 추적
  └─ 다시 미리보기 → 만족 시 내보내기
```

## 6. 상태 다이어그램

```
Init ──▶ ModeSelect ──▶ Loaded ──▶ Segmented ──▶ Selected ──▶ [RangeSet]
                                                                   │
                                                                   ▼
                                       ┌───────────── Tracking ◀──┘   (GIF 모드)
                                       │              │   ▲
                            ReShotInit ┘              ▼   │ 부분 재추적
                            (컷 재매칭)          Preview │
                                                   │     │
                                                   ├─ Correction ┘
                                                   │
                                                   ▼
                                                Exported
```

이미지 모드는 `Selected → [RangeSet] → Preview → Exported` (추적 없음).
