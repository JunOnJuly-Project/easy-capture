# ADR 0002 — GUI 프레임워크로 PySide6 채택

- 상태: 채택
- 날짜: 2026-05-27

## 맥락
프레임 위 클릭으로 오브젝트 선택, 범위 박스 드래그, 타임라인 시작/끝 핸들 드래그 등 **정밀한 인터랙티브 캔버스**가 핵심이다.

## 결정
**PySide6**(Qt, LGPLv3)를 채택. `QGraphicsScene`/`QPainter` 로 캔버스·오버레이·드래그를 저수준 제어할 수 있고, 네이티브 데스크톱 경험과 워커 스레드(QThread) 연동이 우수하다.

## 대안
- Gradio/Streamlit(로컬 웹): 개발은 빠르나 박스 드래그·정밀 캔버스 제약.
- PyQt6: 기능 동등하나 GPL/상용 라이선스 분리 → LGPL 인 PySide6 선호.

## 결과
- 추적/업스케일은 워커 스레드, UI 는 시그널로 진행률·ETA 수신([아키텍처](../architecture.md) §5).
- LGPL 준수를 위해 동적 링크 배포.
