## 1. 배경 및 목적

뮤직비디오·직캠 등 동영상에서 **특정 오브젝트(인물/사물)를 중심으로 자동 크롭**하여 스크린샷(짤)과 GIF(움짤)를 쉽게 만드는 **로컬 데스크톱 프로그램**.

기존 방식의 문제:
- 영상 편집 툴로 특정 멤버를 프레임마다 수동 추적·크롭하는 것은 노동집약적이다.
- 군무·빠른 컷 전환에서 한 인물만 따라가며 잘라내는 작업은 반복적이고 오차가 크다.

목표: **클릭 한 번으로 오브젝트를 지정** → 프로그램이 추적·크롭·내보내기를 자동화하여, 비전문가도 단시간에 결과물을 얻는다.

## 2. 대상 사용자

| 구분 | 설명 | 기대 수준 |
|---|---|---|
| **Primary** | 영상 편집 경험이 있는 팬 콘텐츠 제작자 (직캠·움짤 업로더) | 기본 옵션 이해 가능, 품질·종횡비에 민감 |
| **Secondary** | 일반 덕후 (비개발자) | 클릭 기반 단순 흐름·합리적 기본값 필요 |

모두 **로컬 PC**에서 자신의 영상 파일을 다룬다. (클라우드 업로드 없음)

## 3. 기능 요구사항 (FR)

| ID | 요구사항 | 우선순위 | MVP |
|---|---|---|---|
| FR-01 | 시작 화면에서 **이미지(짤) / GIF(움짤) 모드**를 먼저 선택 | 필수 | ✅ |
| FR-02 | 동영상(mp4/mov/mkv 등) 및 이미지 파일 입력, **입력 검증**(코덱·해상도) 후 경고 | 필수 | ✅ |
| FR-03 | 동영상에서 **특정 시점 프레임 선택**(타임라인 탐색) | 필수 | ✅ |
| FR-04 | 선택 시점에서 **자동 검출(Grounding DINO) + 세그멘테이션(SAM2)** 실행, 후보를 박스·라벨로 표시 | 필수 | ✅ |
| FR-05 | **동일 프레임 다중 인물 구분**: 후보를 각각 라벨링 → 사용자가 하나를 클릭해 추적 대상 1개 확정. 다른 멤버는 세션 재사용으로 순차 처리 | 필수 | ✅ |
| FR-06 | 추적 대상 주변 **크롭 범위(가로×세로) 입력** + 종횡비 잠금 프리셋(1:1 / 9:16 / 16:9 / 자유) | 필수 | ✅ |
| FR-07 | **SAM2 video predictor 기반 추적**: 선택 오브젝트를 후속 프레임에 자동 전파 | 필수 | ✅ |
| FR-08 | **샷 경계 재추적**: 컷 전환 자동 감지 후 동일인 재매칭하여 추적 재개 | 필수 | ✅ |
| FR-09 | **occlusion 대기/재등장**: 오브젝트 소실 시 사용자 지정 대기 시간까지 재등장 대기 후 재추적 | 필수 | ✅ |
| FR-10 | **갭 채우기 3방식**(컷 / 배경 계속 / 프리즈), 기본=컷 | 필수 | ✅ |
| FR-11 | **수동 교정**: 미리보기 중 잘못된 구간에서 재선택 → 그 지점부터 부분 재추적(이전 구간 보존) | 필수 | ✅ |
| FR-12 | GIF 모드: **타임라인 드래그로 시작/끝 구간 지정** + FPS·대기시간·갭방식·팔레트 품질 설정 | 필수 | ✅ |
| FR-13 | 출력: **PNG/JPG(정지)**, **GIF(8비트 인덱스 최대 256색 동적 팔레트·디더·파일크기 예측)**, **MP4(짝수해상도·yuv420p·원본 오디오 패스through·VFR→CFR 정규화)** | 필수 | ✅ |
| FR-14 | **이미지 업스케일링(옵션)**: Real-ESRGAN / SwinIR 중 선택, 2x/4x | 선택 | ✅ |
| FR-15 | 결과 **미리보기** 후 내보내기 | 필수 | ✅ |
| FR-16 | 같은 구간 **다중 멤버 배치 일괄 처리** | 후순위 | ⛔ v1.1 |
| FR-17 | **SNS 플랫폼별 자동 압축/포맷 변환 프리셋** | 후순위 | ⛔ v1.1 |

## 4. 비기능 요구사항 (NFR)

| ID | 요구사항 | 목표 |
|---|---|---|
| NFR-01 | **로컬 실행** | 외부 서버 의존 없음. 모델은 최초 1회 다운로드 후 캐시 |
| NFR-02 | **디바이스 자동 대응** | CUDA 자동 감지 → 없으면 CPU 폴백 + 경고 |
| NFR-03 | **성능 KPI** (측정가능, PoC 확정) | GPU(RTX 3060급) 추적 ≥ 10fps / CPU 1~2fps. 처리 예상 시간 UI 표시 |
| NFR-04 | **GIF 생성 시간** | 480p·5초 구간 기준 GPU ≤ 30초 (목표, PoC 확정) |
| NFR-05 | **응답성** | 추적·업스케일은 워커 스레드에서 수행, UI 비블로킹 + 진행률·ETA 표시 |
| NFR-06 | **지원 OS** | Windows 우선(개발 환경), macOS/Linux 차순위 (상세는 [리소스](resources.md) 플랫폼 매트릭스) |
| NFR-07 | **메모리 안전성** | 전체 영상 디코드 금지, 선택 구간만 스트리밍. 대용량/4K 경고 |
| NFR-08 | **라이선스** | 전 구성요소 상업적 사용 가능(Apache/LGPL/BSD). basicsr 검증 결과는 [리소스](resources.md)에 기록 |

## 5. 범위 (Scope)

### In Scope (MVP v1.0)
이미지/GIF 모드 분리, 자동검출+클릭 단일 오브젝트 추적, 샷 경계 재추적, occlusion 대기·갭 채우기 3방식, 수동 교정, centroid 크롭(떨림완화·종횡비 잠금), PNG/JPG/GIF/MP4(오디오 포함) 출력, 업스케일 2종 옵션, 디바이스 자동 감지.

### Out of Scope (백로그 v1.1+)
다중 멤버 배치 일괄 처리, SNS 플랫폼 자동 프리셋, pose/appearance 기반 고급 re-ID, 직캠 광각 왜곡 보정, 업스케일 temporal smoothing(플리커 저감), simple/advanced 온보딩 마법사.

> Out 항목도 **아키텍처는 확장 대비**한다(트랙 관리·종횡비 IF·업스케일러 공통 IF). 자세한 경계는 [계획서](plans/easy-capture-plan.md)의 MVP In/Out 표 참조.

## 6. 제약 및 가정

- **제약**: GPU 부재 시 추적·업스케일 속도 현저히 저하(실시간 불가). 저화질(<480p)·직캠 광각 영상은 품질 보장하지 않음. **GIF 출력은 8비트 팔레트(BT.709→sRGB 톤매핑)로 고채도 색에서 약간의 색감 손실이 발생할 수 있다**(고품질 필요 시 MP4 권장).
- **가정**: 사용자는 자신이 권리를 가지거나 정당하게 이용 가능한 영상을 입력한다.
- **저작권 고지**: 본 프로그램으로 생성한 콘텐츠의 저작권·이용 책임은 사용자에게 있다. 원본 영상의 저작권 및 배포 플랫폼 약관 준수는 사용자 의무다. (앱 시작 시 고지)

## 7. 측정 가능한 수용 기준 (Acceptance Criteria)

| ID | 기준 | 측정 방법 |
|---|---|---|
| AC-01 | 군무 클립에서 선택 인물 **추적 유지율 ≥ 80%** | PoC 테스트셋 프레임별 IoU 평가 |
| AC-02 | 지정 대기시간 내 **occlusion 후 재추적 성공 ≥ 70%** | PoC 시나리오 통과율 |
| AC-03 | **컷 전환 재매칭 성공 ≥ 70%** | PoC 빠른 컷 샘플 |
| AC-04 | 수동 교정 시 **이전 성공 구간 보존**(재처리 없음) | 동작 확인 |
| AC-05 | MP4 출력에 **원본 오디오 동기 포함**(VFR 포함), 실패 시 무음 폴백+고지 | 출력 검수 |
| AC-06 | GPU 환경 추적 **≥ 10fps**, 처리 예상 시간 UI 표시 | 벤치마크 |
| AC-07 | GIF 출력 시 **파일크기 예측·팔레트 품질 선택** 동작 | UI 확인 |
| AC-08 | **VFR 소스 → CFR 정규화** 시 프레임/오디오 어긋남 ≤ 1프레임 | 출력 검수(PoC H4) |

> 수치는 PoC([poc-plan.md](poc-plan.md))로 최종 확정한다.
