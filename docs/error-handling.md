# easy-capture — 에러 / 엣지케이스 / 폴백 명세

> 최종 업데이트: 2026-05-27 · 관련: [유즈플로우](use-flow.md) · [데이터플로우](data-flow.md)
> 모든 사용자 대면 메시지는 한국어. 복구 가능한 경우 폴백을 우선 제시한다.

---

## 1. 입력 단계

| 상황 | 감지 | 처리 |
|---|---|---|
| 미지원 코덱 | ffprobe 파싱 실패 | 거부 + "지원하지 않는 형식입니다. MP4(H.264) 권장" |
| 손상/잘린 파일 | 디코드 예외 | 거부 + 안내, 부분 재생 시도 옵션 |
| 저해상도(<480p) | 해상도 검사 | 경고(계속 진행 가능) + 업스케일 권유 |
| 직캠(광각/세로 팬캠) | 메타·종횡비 | "직캠은 화면 모서리 추적 정확도가 낮을 수 있어요" 안내. 광각 왜곡 보정은 v1.1 |
| 4K 등 초고해상도 | 해상도 검사 | 메모리 경고 + 처리 해상도 다운 옵션 제안 |
| VFR 영상 | ffprobe 의 avg/r_frame_rate 불일치 | 정보 표시, PTS 기반 처리로 자동 전환 |

---

## 2. 세그멘테이션 / 선택

| 상황 | 처리 |
|---|---|
| 검출 0건(사람/사물 없음) | "자동 검출 없음 — 화면을 직접 클릭해 지정하세요" (SAM2 직접 프롬프트) |
| 후보 과다 | 신뢰도 상위 N개만 표시 + "더 보기" |
| 잘못된 클릭(배경) | 마스크 미생성 시 재클릭 안내 |

---

## 3. 추적 단계

| 상황 | 처리 |
|---|---|
| 추적 신뢰도 저하 | 해당 구간 표시 + "확인이 필요해요" → 수동 교정 유도 |
| 컷 재매칭 실패(점수<임계값) | "컷 이후 같은 대상을 못 찾았어요" → 사용자 확인/수동 교정 |
| occlusion 대기 초과 | 구간 종료 + 갭 정책 적용 안내 |
| identity switch 의심 | 신뢰도·위치 급변 감지 시 경고 마커 |

---

## 4. 업스케일

| 상황 | 폴백 순서 |
|---|---|
| VRAM 부족 | ① 타일 크기 축소 → ② 해상도 다운 → ③ CPU 처리(느림 경고) |
| CPU 전용 | 처리 전 "수 분 소요 가능" 경고, 진행률·ETA 표시 |
| **프레임 간 플리커** | 고지: "2x/4x 프레임 단독 처리라 약간의 떨림이 보일 수 있어요. 프레임 일관성 보정(temporal smoothing)은 v1.1 예정" |

---

## 5. 출력 / 인코딩

| 상황 | 처리 |
|---|---|
| VFR 소스 MP4 출력 | 지정 FPS **CFR 로 정규화**(PTS 기반 프레임 배치) + 오디오 PTS 동기 |
| **오디오 동기 실패(특히 VFR, 어긋남>1프레임)** | **무음 MP4 로 폴백 + "오디오 동기에 실패해 무음으로 저장했어요" 고지** |
| 홀수 해상도 | 자동 짝수 정렬(yuv420p 요구), 사용자에게 정렬 결과 표시 |
| GIF 용량 초과(예: X 15MB) | 경고 + 팔레트 품질↓/FPS↓/구간↓ 또는 MP4 전환 제안 |
| GIF 색감 손실 | 고지: "GIF 는 256색 한계로 고채도 색이 약간 달라질 수 있어요. 원색 보존이 중요하면 MP4 를 권장해요" |
| 저장 경로 권한 오류 | 다른 경로 선택 유도 |

---

## 6. 모델 / 환경

| 상황 | 처리 |
|---|---|
| 모델 다운로드 실패 | 재시도(기본 3회·지수 백오프) → 실패 시 "네트워크 확인 후 재시작" 안내 |
| 오프라인 + 캐시 없음 | 다운로드 필요 모델·용량 안내 |
| CUDA 미감지 | CPU 폴백 + 성능 경고 1회 표시 |
| 디스크 부족(캐시) | 필요 용량 안내, 캐시 경로 변경 옵션 |

---

## 7. 공통 원칙

- 복구 가능한 오류는 **차단 대신 폴백 + 고지**.
- 비가역 동작(덮어쓰기) 전 확인.
- 모든 워커 작업은 **취소 가능**하고, 취소 시 부분 산출물을 정리한다.
